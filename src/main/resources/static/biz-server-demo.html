业务系统A（systemA / 电子书网站）- 消息推送设置页面

这个页面模拟“业务系统 Server 端如何接入你的消息推送平台”。流程就是：引入 SDK → new 服务对象（SM）→ set 推送配置 → 推送消息。业务系统只做配置和发送，客户端只做登录上线后被动接收。

引用 server sdk 配置推送方式 选择目标 Client 平台负责把消息推过去

一、业务系统 Server 接入 SDK（核心三步）

Java 示例（引入 -> new 对象 -> set/推送）

// 1) 引入 SDK
import com.wmpp.sdk.server.SMSetup;

// 2) new 出服务对象（固定平台地址 + 业务系统身份）
SMSetup sm = new SMSetup();
sm.setServerUrl("http://localhost:8080");
sm.setAppKey("systemA");
sm.setSecretKey("systemA-secret");

// 3) 设置并发起推送
sm.broadcast("新书上架通知");

Python / 其他语言模式相同

# 1) 引入 SDK
from wmpp_server_sdk import SMSetup

# 2) 创建对象
sm = SMSetup()
sm.set_server_url("http://localhost:8080")
sm.set_app_key("systemA")
sm.set_secret_key("systemA-secret")

# 3) 设置目标并推送
sm.set_target_type("USER")
sm.set_target_user_id("1001")
sm.set_message("您关注的书已更新")
sm.push()
关键点：WMPP 是独立推送平台，不是业务系统本身。业务系统 Server 只要接入 SDK、配置 appId/appSecret 与目标用户，即可完成推送；不限制业务系统语言和架构。
