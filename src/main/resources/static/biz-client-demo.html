业务系统 Client 上线接收页

这个页面模拟业务系统自己的终端客户端。终端只需要：① 引入 client sdk ② 用户登录时调用 connect/login ③ 被动监听消息。后续消息由 WMPP 平台主动推送过来。它不是聊天窗口，只负责“被动接收消息”。

业务系统终端 引入 SDK 后登录即连接 只接收，不发送 同一系统多个 Client 可同时接收 不同系统消息严格隔离

终端职责非常简单：业务系统用户一登录，客户端立即连接 WMPP 推送平台，然后等待平台把消息主动推送过来。客户端不关心其他业务系统，也不会收到其他系统的消息。

再去业务系统 Server 推送设置页操作：
1) 广播：同系统多个 Client 同时收到；
2) 指定用户：只有目标用户收到；
3) 跨系统隔离：systemA 推送不会到 systemB，反之亦然。
