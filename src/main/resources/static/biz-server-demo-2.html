业务系统B（systemB / 电商网站）- 消息推送设置页面

WMPP 是独立消息推送基础设施，业务系统只接入 SDK。你可以用 JavaScript 或 Python 写业务服务端，接入模式都一样：引入 SDK、创建对象、设置推送参数、发消息。

SDK 接入 单次 / 周期 全部 / 固定用户 / 一组用户 与 systemA 严格隔离

一、Server SDK 接入示例（JavaScript / Python）

JavaScript 示例

import { SMSetup } from "wmpp-server-sdk";

const sm = new SMSetup();
sm.setServerUrl("http://39.107.127.13:8080");
sm.setAppKey("systemB");
sm.setSecretKey("systemB-secret");
sm.setMessage("内容社区热点通知");
sm.setTargetType("GROUP");
sm.setTargetUsers(["3003", "3004"]);
await sm.push();

Python 示例

from wmpp_server_sdk import SMSetup

sm = SMSetup()
sm.set_server_url("http://localhost:8080")
sm.set_app_key("systemB")
sm.set_secret_key("systemB-secret")
sm.set_message("内容服务通知")
sm.set_target_type("ALL")
sm.push()
重点：业务系统技术栈不限。WMPP 只提供统一 API/SDK 服务，业务系统只处理“何时给谁推什么”。
