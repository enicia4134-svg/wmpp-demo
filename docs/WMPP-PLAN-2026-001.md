## 《Web消息推送系统项目计划书》

**Web Message Push Platform (WMPP) — Project Plan**

- **文档编号**：WMPP-PLAN-2026-001  
- **版本**：V1.0  
- **作者**：郑青惟  

---

## 1 项目概述

本项目旨在设计并实现一个基于 Web 的实时消息推送平台，为多个业务系统提供统一的消息推送能力。系统采用微服务架构思想，基于 WebSocket 技术实现客户端长连接维护，并通过 Server-Sent Events（SSE）技术完成实时消息推送。

系统采用容器化部署方式运行于云服务器环境，具备良好的可扩展性与可维护性。

---

## 2 项目目标

- 实现多业务系统接入的统一推送平台
- 实现在线用户实时消息推送能力
- 实现推送节点动态调度机制
- 完成系统云端部署与演示

---

## 3 项目开发阶段划分

### 第一阶段：开发环境搭建

- 完成云服务器采购与系统环境部署，包括操作系统安装、开发环境配置以及容器运行环境搭建
- 完成基础 Demo 编写，验证 WebSocket 长连接与 SSE 推送功能在云服务器环境可稳定运行

### 第二阶段：核心模块开发

- 完成 Pusher Worker 服务开发，实现客户端连接注册、心跳检测与实时消息推送功能
- 完成 Session Registry 模块开发，实现在线连接管理与资源回收机制

### 第三阶段：推送平台构建

- 完成 Push Gateway 与 Scheduler 模块开发，实现推送任务调度与推送节点资源动态分配能力
- 完成 Server SDK 与 Client SDK 开发，实现业务系统推送调用与客户端接收消息功能

### 第四阶段：系统集成与测试

- 完成系统联调测试，验证推送流程完整性与系统稳定性
- 进行功能测试与基础性能测试，分析系统运行日志并优化系统结构

### 第五阶段：论文撰写与答辩准备

- 完成毕业论文撰写，整理系统设计文档与测试报告
- 完成系统部署演示准备与答辩材料制作

## 5 风险分析与应对措施

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 长连接稳定性不足 | 推送可靠性下降 | 分阶段验证、增强日志与监控、增加心跳与回收策略 |
| 调度策略复杂度提升 | 开发延期 | 先 RR 再 LeastConn，抽象策略接口，后续逐步增强 |
| 云端部署环境差异 | 演示失败风险 | Docker 化、提供一键启动脚本与健康检查 |
| SDK 易用性不足 | 接入成本高 | 统一参数与错误码、提供示例与最小接入文档 |
